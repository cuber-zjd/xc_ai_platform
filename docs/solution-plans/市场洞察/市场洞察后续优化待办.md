# Insight 后续优化功能池

文档类型：非阻塞后续优化清单
适用模块：`/insight/*` 市场洞察平台
创建日期：2026-06-09

## 1. 文档目的

本文档集中记录 Insight 当前暂未完成、但不阻塞核心主流程闭环的功能。后续优化时优先从本文档取项，避免散落在对话、阶段计划或单点问题里。

核心主流程指：

1. 数据源配置。
2. 采集与候选情报生成。
3. 候选审核转正式情报。
4. 正式情报维护与素材池。
5. 报告生成、编辑、HTML/PDF/DOCX 通用导出与下载。
6. 后端权限过滤与基础调度闭环。

只要以上主流程可稳定运行，本文档中的功能可以按业务优先级延后建设。但延后功能不得在前端、接口文档或用户培训中被描述为“已支持”。

## 2. 优先级口径

| 优先级 | 含义 |
| --- | --- |
| P0 | 上线前建议补齐，否则会影响错误恢复、用户误解或运营安全。 |
| P1 | 上线后第一轮增强，明显提升交付质量或运营效率。 |
| P2 | 稳定期优化，适合在真实使用反馈后推进。 |
| P3 | 长期能力，不建议近期承诺。 |

## 9. 后续使用规则

1. 每次准备做后续优化时，先从本文档选择编号。
2. 开始修复前，在主上线计划的对应阶段记录本次目标。
3. 完成后更新本文档对应项状态，写入改动文件、验证命令和结论。
4. 若某项被业务决定不做，标记为“暂缓”并说明原因。
5. 未通过验收脚本或人工验收前，不允许把该能力写成“已支持”。
