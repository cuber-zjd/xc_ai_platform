# 市场洞察暂未实现渠道清单

更新时间：2026-06-16

## 处理原则

- 第一版只采集公开可访问内容，不绕过登录、验证码、付费墙、平台风控和版权限制。
- 导入《信息采集需求收集.xlsx》《网址整理.docx》时，能通过“搜索发现 + 正文抓取 + AI筛选”覆盖的渠道会生成数据源。
- 暂不适合自动化的渠道必须保留记录，后续通过官方 API、采购数据、人工导出或企业账号授权方式接入。

## 暂未自动化渠道

| 渠道类型 | 典型渠道 | 暂未实现原因 | 当前替代方案 | 后续建议 |
| --- | --- | --- | --- | --- |
| 电商登录/强风控页面 | 淘宝、天猫、京东、盒马、小红书、亚马逊、Lazada、Shopee | 商品搜索和详情常有登录态、验证码、反爬、地域化价格和动态渲染，直接抓取稳定性低且存在合规风险 | 先通过公开搜索引擎抓取可索引页面、新闻稿、品牌官网和第三方行业报道 | 评估官方开放平台、企业采购账号授权、人工导出 Excel 后导入 |

| 专利库高级检索 | CNIPA 高级检索、WIPO 高级检索 | 高级检索结果页动态复杂，存在验证码和访问频控 | 先用公开搜索发现专利标题、公告、新闻和企业专利动态 | 评估官方数据服务、批量下载或第三方专利数据库 API |
| 付费财经/研报 | Wind、Choice、部分券商研报、付费数据库 | 需要合同授权和账号体系，不应由爬虫绕过 | 采集交易所公告、上市公司公告、公开财经新闻 | 若已有授权，走供应商 API 或人工导出后导入 |
| 政府网站附件深层解析 | 部分 PDF/Word/表格附件 | 站点结构差异大，附件格式不统一，第一版仅保证正文 URL 抓取 | 搜索发现政策页面并抓取可读正文摘要 | 增加附件下载、OCR/PDF 解析和政策字段抽取任务 |

## 已支持的第一版自动化策略

- 企业官网、行业媒体、财经资讯、政府政策、专利公开信息：通过数据源关键词进行搜索发现，抓取公开正文，AI 做摘要、标签、情感、机会和风险提取。
- 公众号：采集公开可访问或搜索引擎已索引文章，不模拟登录。
- 电商：采集公开可访问商品/品牌/行业报道页面，不绕过平台风控。
- 无明确 URL 的企业和渠道：先生成搜索型数据源，用企业名称、课题和采集需求作为关键词。

## 运行期补充机制

- 数据源导入接口会把无法识别或不适合自动化的渠道返回到 `unsupported_channels`。
- 期初跑数时若某数据源连续失败，应在本文件追加失败渠道、失败原因、替代方案和后续接入建议。
