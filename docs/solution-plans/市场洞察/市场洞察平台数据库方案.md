# 研发营销市场洞察平台数据库方案

INSIGHT DATABASE PLAN

这份方案用于判断子平台能否真正支撑“采集、清洗、AI 分析、人工审核、权限分发、情报池选材、企业/行业画像、自定义报告、自定义图表、订阅通知、审计反馈”的业务闭环。方案按 PostgreSQL + SQLModel 设计，优先贴合当前 AI Platform 的表命名、基础字段和 JSONB 使用习惯。

数据库**PostgreSQL**

ORM**SQLModel**

业务域**insight**

更新时间**2026-06-01**

## 1. 设计原则

### 可追溯

情报必须能追到原始来源、采集任务、AI 分析版本、人工审核人和最终报告引用。

### 可运营

数据源、采集任务、标签字典、订阅规则和质量反馈都应入库，不把运营配置写死在代码里。

### 可演进

核心事实字段结构化，AI 输出和来源扩展使用 JSONB，但避免把所有业务对象都塞进 JSON。

所有业务表默认继承项目现有 `BaseDBModel`：`id`、`create_time`、`update_time`、`create_by`、`update_by`、`comment`、`is_deleted`。下文只列各表专属字段。

## 2. 领域拆分与关系总览

**insight_company**企业档案主数据

**insight_intelligence**情报主表，可关联企业、行业、主题、来源、标签、AI 分析

**insight_data_source**数据源配置，驱动采集任务

**insight_task**采集、清洗、AI 分析、报告生成任务

**insight_ai_analysis**AI 结构化分析结果和模型版本

**insight_report**日报、周报、专题报告和版本

**insight_tag**统一标签字典

**insight_visibility_rule**情报可见性规则，控制谁能看什么

```
核心关系：
company 1 - n intelligence
industry/topic 1 - n intelligence
data_source 1 - n intelligence_source
intelligence 1 - n intelligence_source
intelligence n - n tag
intelligence 1 - n ai_analysis
intelligence n - n user/role/dept through visibility_rule
report n - n intelligence
report 1 - n report_chart
user n - n subscription(company / tag / report / query)
task 可关联 data_source / intelligence / report
```

### 7.3 权限判断建议

```
用户可见情报 = 
1. visibility_scope = public
2. owner_user_id = 当前用户
3. visibility_scope = dept 且用户属于授权部门
4. visibility_scope = role 且用户拥有授权角色
5. visibility_scope = assigned 且 insight_visibility_rule 存在 user/role/dept view 授权
6. 超级管理员或情报管理员可按管理权限查看
```

列表接口必须在后端拼接权限条件，不能返回全量情报再交给前端隐藏。否则用户仍可能通过接口看到未授权数据。

### 8.4 报告生成时的素材选择

```
报告素材来源可以组合：
1. 用户从情报池手动勾选：insight_user_intelligence_pool.pool_type = report_candidate
2. 按偏好自动筛选：insight_report_preference.source_filter
3. 按模板章节拉取：insight_report_template.template_schema.sections
4. AI 推荐补充：写入 insight_report_material，selection_source = ai_recommended
5. 人工最终确认：selection_source = manual_selected
```

## 12. JSONB 使用边界

### 适合放 JSONB

- AI 输出：机会点、风险点、影响评估、建议标签。
- 采集配置：选择器、解析规则、请求头引用。
- 原始来源快照：抓取元数据、解析器版本。
- 报告内容块：章节、素材引用快照。
- 模板结构：章节占位符、默认模块、导出样式。
- 图表配置：维度、指标、颜色、显示规则。
- 用户偏好：报告语气、篇幅、默认筛选条件。

### 不建议放 JSONB

- 企业、情报、标签、状态、时间等高频筛选字段。
- 用户订阅对象、报告素材关系等需要 JOIN 的关系。
- 情报可见性、用户情报池、报告图表列表等需要单独授权或排序的关系。
- 审核状态、任务状态等需要强约束和统计的字段。
- 敏感密钥明文。
