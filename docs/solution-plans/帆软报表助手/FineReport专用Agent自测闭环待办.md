# FineReport 专用 Agent 自测闭环待办

## 背景

当前小驰已经具备直接读取和修改 CPT XML、生成待应用修改项、版本归档和 FineReport 预览的基础能力。后续需要从“一次性生成修改项”升级为“能观察、修改、验证、失败后继续修复”的 FineReport 专用 Agent。

## 待办方向

1. 引入受控 ReAct 任务循环：理解需求、读取上下文、调用工具、生成补丁、写入版本、预览验证、失败后继续定位。
2. 增加业务验证工具：报表非空、预览可打开、参数栏存在、下拉框有候选项、选择参数后查询可刷新、数据集 SQL 可执行。
3. 增加参考报表检索工具：按控件、数据集、填报、脚本、样式等能力从已有 CPT 中检索可用写法。
4. 增加错误反馈再修复：将 FineReport 预览错误、浏览器 DOM 状态、CPT 差异和控制台错误作为下一轮修复上下文。
5. 增加真正的流式接口：后端提供小驰 SSE 输出，前端展示工具进度、局部回答和待应用修改项就绪事件。
6. 优化历史经验检索：成功和失败案例只在相关场景按需检索，不直接写入系统提示词。
7. 保持版本控制与确认门：中高风险修改必须确认；每次写入 CPT 仍通过平台结构版本和文件版本归档。

## 验收标准

- 用户反馈“预览空白”“下拉没有数据”“参数没生效”等问题时，小驰能自动读取相关 CPT 片段和参考报表，自行生成修复补丁并完成预览验证。
- 用户无需理解 XML 或内部 JSON，只看到自然语言的修改范围、风险提示、确认按钮和验证结果。
- token 使用保持按需读取，不把完整 CPT、完整历史会话或大量样例行默认注入模型上下文。
