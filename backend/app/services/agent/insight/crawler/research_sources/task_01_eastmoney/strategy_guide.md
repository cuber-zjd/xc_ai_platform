# 东方财富网爬虫运行策略与防封指南 (Strategy & Anti-Blocking Guide)

本文档阐述了针对东方财富网进行稳定、长效数据抓取的方案设计与防封限速规范，作为本任务的运营标准。

---

## 1. 核心抓取策略 (爬取逻辑)

东方财富网（特别是股吧和搜索板块）对自动化爬虫的拦截点主要在于：
*   **高频 IP 访问限制**：对短时间内的密集高频访问，WAF 会触发验证码拦截，阻断接口返回。
*   **请求链头校验**：动态 API 接口对 `Referer`、`User-Agent` 等请求头进行了格式校验。

为了避开这些防爬限制，我们将使用 **Playwright 网络拦截监听** 机制：
1.  **首屏路由跳转**：直接通过 Playwright 进入东方财富的搜索结果页或股吧搜索页。
2.  **API 自动化匹配**：绑定全局的 `page.on("response", callback)`，对下发的 AJAX 请求进行实时过滤。一旦发现包含数据列表的接口，直接在内存中保存其 JSON 响应体。
3.  **多轮滚动与翻页**：利用 Playwright 执行滚动或模拟点击下一页，促使页面内部自动触发接口数据调用。

---

## 2. 持续性运行与防封规范

### A. 频次与步进限速
*   **步进式休眠**：在抓取多个关键词时，词与词之间至少要间隔 8~12 秒，以避免触发 WAF 的高频防爬警戒线。
*   **分批机制**：建议每天的爬取任务按照每 10 个词划分为一组，两组之间至少保持 15 分钟的静默冷却期。

### B. 伪装指纹规范
*   **不要强行篡改 webdriver 特征**：高级风控盾会对 `Object.defineProperty` 劫持特征进行逆向识别。必须使用 Playwright 默认的 `--disable-blink-features=AutomationControlled` 并带上真实 UA。
*   **设置有效的 Referer**：在请求被拦截并试图以原生 API 重发时，必须配置与真实页面跳转相匹配的 `Referer` 字段（如 `"https://guba.eastmoney.com/"`）。

### C. 验证码熔断
*   若在监听器或页面文本中频繁检测到 `verify`、`slider`、`验证码` 或 `滑动验证` 等拦截阻断信号，说明 IP 已经被暂时拉入限制组，**必须立即终止运行（熔断保护）**，严禁进行高频重试攻击，否则将面临 IP 长期甚至永久被墙的危险。
