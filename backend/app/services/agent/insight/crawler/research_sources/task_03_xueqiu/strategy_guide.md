# 雪球网爬虫运行策略与防封指南 (Strategy & Anti-Blocking Guide)

本文档阐述了针对雪球网进行稳定、长效数据抓取的方案设计与防封限速规范，作为本任务（以及后续类似爬虫任务）的通用运营标准。

---

## 1. 核心抓取策略 (爬取逻辑)

雪球网在未登录状态下的搜索服务受阿里云 WAF 安全防护，采用直接 API 请求或常规无头浏览器访问会被 100% 阻断。我们的核心抓取采用 **Playwright 异步事件拦截 + 主动 Reload 重载** 的双轨机制：

1.  **Session 预建立**：先访问雪球主页 `https://xueqiu.com/` 获取服务器下发的初始游客 Cookies（特别是 `xq_a_token`）。
2.  **触发 WAF 挑战**：转入搜索页面后，WAF 立即拦截并返回人机挑战 HTML，浏览器在后台静默执行其安全 JS，计算设备指纹。
3.  **主动重载 (Reload)**：等待 6~8 秒后，WAF 已将凭证写入浏览器 Cookie。此时执行 `page.reload()`，浏览器带上验证凭证重新发起请求，WAF 随之放行 API 数据，直接在 `response` 监听中截获合法的 `status.json` JSON 数据包。
4.  **复用 Context 抓取**：对于多个关键词，在同一个 `BrowserContext` 会话中循环处理。第一个词通过 WAF 挑战后，后面的词无需再次等待，可直接通过校验获取数据。

---

## 2. 持续性运行与防封规范

为了保证本方案长期运行不被拉黑，必须在部署和调度时严格执行以下规范：

### A. 频次控制与定时分批 (Time Slicing)
*   **严禁单次连续大量爬取**：如需爬取近百个关键词，**严禁一次性循环执行完毕**。WAF 检测到单 IP 的高频调用会从“静默 JS 挑战”升级为“强力滑动验证”，导致脚本彻底卡死。
*   **定时分组运行（推荐）**：
    - 将每日关键词列表分为多组，每组包含 **10 个** 关键词。
    - 使用系统的定时任务（如 Windows 计划任务或 Crontab），**每 15~20 分钟** 启动一次脚本，执行其中一组。
*   **步进式休眠**：在每组词的抓取中，每个词之间加入随机休眠，例如：
    ```python
    time.sleep(random.uniform(8, 15))
```

### B. 代理 IP 轮换 (Proxy Rotation)
当单日抓取规模上升到 **几百个关键词以上** 时，单 IP 将无法支撑，必须引入动态代理 IP 池：
*   在 `browser.new_context(proxy=...)` 中，为每次运行随机分配一个优质代理 IP。
*   每抓取 **5~10 个关键词** 后，主动关闭浏览器并更换代理 IP 重新发起新会话。

### C. 特征隐藏与指纹防御
*   **严禁强行篡改 WebDriver 变量**：不要使用 `Object.defineProperty(navigator, 'webdriver'...)` 等自定义 JS 代码强行将 webdriver 设为 undefined。阿里云等高级 WAF 盾能轻易通过检测属性描述符劫持来直接识别爬虫。
*   **使用 Playwright 默认去特征化**：保持 `--disable-blink-features=AutomationControlled` 并搭配主流真实的 `User-Agent`，使指纹看起来像一个普通的现代 Chrome 浏览器。

### D. 异常熔断机制 (Breaker)
*   如果在 WAF 挑战或 Reload 阶段，拦截器获取到的响应体频繁出现 `geetest`（极验）或 `verification`（验证码）等阻断字样，说明当前 IP 已被风控判定。
*   **处理规范**：脚本必须**立即抛出异常并终止运行（熔断）**，暂停后续词的抓取，并在一定时间（如 1 小时）后重试，避免继续强攻导致 IP 被彻底封杀。
