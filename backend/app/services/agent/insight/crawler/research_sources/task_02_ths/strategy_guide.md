# 同花顺网爬虫运行策略与防封指南 (Strategy & Anti-Blocking Guide)

本文档阐述了针对同花顺网（包括同花顺财经搜索及问财平台）进行稳定、长效数据抓取的方案设计与防封限速规范，作为本任务的运营标准。

---

## 1. 核心抓取策略 (爬取逻辑)

同花顺网的核心防爬机制围绕 **`hexin-v` 动态 Cookie（Cookie 中的 `v` 值）** 展开。直接通过 Python `requests` 访问接口通常会被返回 `403` 或阻断。

我们的核心抓取方案继续采用 **Playwright 异步拦截**：
1.  **浏览器环境注入**：使用 Playwright 打开搜索或股吧的主页，利用浏览器原生的 JS 运行环境执行同花顺网页上的加密签名脚本。
2.  **获取动态凭证**：在加载完搜索页面后，底层网页会自运算出最新的 `hexin-v` 参数并自动附带在请求中。
3.  **接口监听拦截**：拦截搜索 API 的返回，由于 Cookie 由浏览器自动维持，数据接口能够 100% 顺畅通行返回 JSON 数据。

---

## 2. 持续性运行与防封规范

### A. 频次控制与定时分批
*   **组队间隔**：每个关键词的抓取必须维持至少 10~15 秒的合理冷却，在组与组之间，定时任务需有 15 分钟以上的静默。
*   **会话复用**：在单个批次的多个关键词中，必须复用同一个 `BrowserContext` 或 `page` 会话以重复使用已有 `hexin-v` Cookie，严禁每个词重新起一个新浏览器，以免频繁触发 WAF 防指纹机制。

### B. 防特征拉黑
*   **不要篡改 webdriver 特征**：同花顺盾会审查属性描述符。只用常规去特征启动参数并配置真实 UA。
*   **防止异常重试**：若接口返回的内容中含有验证码跳转或提示阻断，脚本必须触发熔断保护立即退出，不可死循环重试。
